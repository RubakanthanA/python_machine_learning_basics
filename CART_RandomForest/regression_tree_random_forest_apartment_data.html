Fit the regression tree model¶

In [ ]:
# Create decision tree regressor object - > Modell wird vorbereitet
# Erstellung eines Entscheidungsbaum-Regressors
# random_state=20: Steuert die Zufallszahl, die die Ergebnisse beeinflusst. immer die gleichen Ergebnisse zu erhalten. (frei wählbar)
# max_depth=3: max_depth=3: Begrenzt die Tiefe auf 3 Ebenen. Dies verhindert Überanpassung (Overfitting), 
# indem die Komplexität des Baums eingeschränkt wird. Eine geringere Tiefe sorgt dafür, dass der Baum allgemeiner bleibt und nicht zu viele Details lernt.

reg = DecisionTreeRegressor(random_state=20, max_depth=3)

# Train decision tree regressor - Trainieren des Modells: 
# Das fit-Verfahren trainiert den Entscheidungsbaum anhand der Trainingsdaten (X_train und y_train).
# - X_train: Eingabedaten, also die Merkmale, die zur Vorhersage der Zielvariable verwendet werden.
# - y_train: Zielwerte (hier vermutlich die Preise), die das Modell lernen soll.
# Beim Training lernt der Baum Muster in den Daten und erstellt eine Baumstruktur mit Entscheidungsregeln, die die Zielvariable (hier y_train) möglichst gut vorhersagen.

reg = reg.fit(X_train, y_train)

# Predict the response for test dataset - Vorhersagen erstellen
# X_test enthält die Merkmale der Testdaten, ohne die Zielwerte.
# y_pred ist das Ergebnis der Vorhersage und enthält die vom Modell vorhergesagten Zielwerte (hier vermutlich Preise) für die Testdaten.

y_pred = reg.predict(X_test)

Calculate coefficient of determination (R-squared)¶

In [18]:
# Calculate coefficient of determination
# r2_score ist eine Funktion Bestimmtheitskoeffizienten 
# misst wie gut die orhergesagten Werte (y_pred) mit den tatsächlichen Werten (y_test) übereinstimmen
# R2 Wert liegt zwischen 0 und 1 
#   R2: 1 (Wert) bedeutet, dass das Modell die Daten perfekt erklärt
#   R2: 0 Bedeutet, dass das Modell die Daten gar nicht erklärt.
# round(..., 4): Wert auf vier Dezimalstellen zu runden.
# print('R-squared:', ...): Ergebnis 

# Beispiel und Intepretation:
# - R-squared: 0.85 (nah an 1): Das Modell erklärt 85 % der Varianz der Zielvariablen y, was auf eine gute Anpassung hinweist.
# - R-squared: 0.3 (näher an 0): Das Modell erklärt nur 30 % der Varianz, was bedeutet, dass es die Daten weniger gut erklärt.

print('R-squared:', round(r2_score(y_test, y_pred), 4))

R-squared: 0.4502

Fit the Random Forest Regression¶

In [ ]:
# n_estimators anpassen und schauen, wie sich das Ergbniss R-squared verändert
# Erklärung: Random Forest Regressor-Modell, um zu sehen, wie sich die Anpassung des Hyperparameters n_estimators auf das Ergebnis des Modells auswirkt, 
# insbesondere auf den Bestimmtheitskoeffizienten 𝑅2

# RandomForestRegressor: Erstellt ein Random-Forest-Modell zur Vorhersage kontinuierlicher Zielvariablen (wie z. B. den Preis).
# n_estimators=1000: Legt die Anzahl der Bäume Fest. In diesem Beispiel auf 1000. 
# - Mehr Bäume können zu besseren Ergebnissen führen, da sie die Robustheit des Modells erhöhen, aber sie erhöhen auch die Rechenlast.

# max_depth=10: Begrenzt die Tiefe jedes Entscheidungsbaums auf 10 Ebenen.
# -Dies verhindert Überanpassung (Overfitting), da tiefere Bäume oft dazu neigen, zu viele Details zu lernen und möglicherweise zu stark auf die Trainingsdaten angepasst sind.
# random_state=5: Sorgt für Reproduzierbarkeit. Wenn der gleiche Wert für random_state verwendet wird, wird das Modell bei jedem Durchlauf gleich initialisiert.

# reg_rf.fit(X2_train, y2_train): Random-Forest-Modell wird trainiert
# - (d. h., es lernt die Beziehung zwischen den Eingabemerkmalen X2_train und der Zielvariable y2_train).
# Das Modell erstellt eine Vielzahl von Entscheidungsbäumen, die durch zufällige Unterstichproben der Daten erstellt werden. 
# Diese Bäume werden dann kombiniert, um eine stabilere und robustere Vorhersage zu erzielen.

reg_rf = RandomForestRegressor(n_estimators=1000, 
                               max_depth=10, 
                               random_state=5)
reg_rf.fit(X2_train, y2_train)

# Calculate coefficient of determination (R-squared) - Berechnung des Bestimmtheitskoeffizienten R2
# Berechung R2 Wert für die Testdaten (X2_test und y2_test), was eine Maßzahl für die Modellgüte ist.
# Der R2 Wert gibt an, wie gut das Modell die Varianz der Zielvariable erklärt:
# - R2 = 1: Perfekte Modellanpassung; das Modell erklärt die gesamte Varianz.
# - R2 = 0: Keine Verbesserung gegenüber dem Durchschnittswert der Zielvariablen.
# - R2 = <0: Das Modell erklärt die Varianz schlechter als der Mittelwert der Zielvariablen.
# round(..., 4): Rundet das Ergebnis auf vier Dezimalstellen für eine übersichtliche Darstellung


print('R-squared:', round(reg_rf.score(X2_test, y2_test), 4))


# Experiment: Anpassen von n_estimators
# -Erhöhung von n_estimators: Mehr Bäume können die Vorhersagegenauigkeit erhöhen, da das Modell dadurch mehr Variationen in den Daten erfasst.
# -Reduzierung von n_estimators: Weniger Bäume verringern die Rechenzeit, können aber auch die Modellgenauigkeit senken.

R-squared: 0.5468

# Extrahieren der Spaltennamen
# - Hier werden die Spaltennamen der Trainingsdaten X2_train in die Variable cols gespeichert.
#     Diese Namen werden später verwendet, um die Bedeutung der einzelnen Merkmale in der Ausgabe zu identifizieren.

cols = X2_train.columns

# Derive feature importance from random forest - Berechnung der Feature Importance im Random Forest
# reg_rf.feature_importances_: Dies gibt die Bedeutung (Importance) jedes Merkmals im trainierten Random Forest Modell reg_rf als Array zurück.
# - Ein höherer Wert bedeutet, dass dieses Merkmal mehr Einfluss auf die Modellvorhersagen hat.
# std:Berechnet die Standardabweichung der Feature Importances für jedes Merkmal, Standardabweichung hilft die Stabilität einzuschätzen
# - Eine hohe Standardabweichung bedeutet, dass die Wichtigkeit des Merkmals je nach Baum stark variiert.
# indices = np.argsort(importances)[::-1]: 
# - Sortiert die Indizes der Merkmale in absteigender Reihenfolge der Importance-Werte.
# - [::-1] dreht die Reihenfolge um, sodass die Merkmale mit der höchsten Bedeutung zuerst erscheinen.

importances = reg_rf.feature_importances_
std         = np.std([tree.feature_importances_ for tree in reg_rf.estimators_], axis=0)
indices     = np.argsort(importances)[::-1]

# Print col-names and importances-values - Ausgeben der Merkmalsnamen und ihrer Bedeutungswerte
# cols[indices]: Gibt die Namen der Merkmale in der Reihenfolge ihrer Bedeutung aus.
# importances[indices]: Gibt die Importance-Werte der Merkmale in der gleichen Reihenfolge aus.
# -  beiden Zeilen zeigen die wichtigsten Merkmale zuerst an, sodass schnell ersichtlich ist, welche Merkmale für das Modell am bedeutendsten sind.

print( cols[indices] )
print( importances[indices] )

# Barplot with feature importance - Erstellung eines Balkendiagramms der Feature Importance
# pd.DataFrame({'features': cols, 'importances': importances}): Erstellt ein DataFrame df_fi, das die Merkmale und deren Bedeutung enthält.

# f_fi.sort_values('importances', inplace=True): edrigsten Bedeutung zuerst und die mit der höchsten Bedeutung zuletzt stehen.
# kann auch anders rum gemacht werden:
# ascending=False: Sortiert in absteigender Reihenfolge. höchsten Importance oben stehen.


# plot(kind='barh', y='importances', x='features')
# -  horizontales Balkendiagramm (kind='barh'),  y-Achse (Importances) auf der x-Achse angezeigt werden.


df_fi = pd.DataFrame({'features':cols,'importances': importances})
df_fi.sort_values('importances', inplace=True)
df_fi.plot(kind='barh', 
           y='importances', 
           x='features', 
           color='darkred', 
           figsize=(6,3))
